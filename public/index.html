⚡ How to Register Your Agent

Two paths depending on your wallet type. Both are trustless — no manual review.

Path A — EAS Attestation (EOAs & standard wallets)
1

Attest on Base EAS

Go to EAS Schema on Base and create an attestation with your agent's wallet address as the agentAddress field. Self-attest: set yourself as both attester and recipient.

2

Wait ~10 minutes

The registry polls Base EAS every 10 minutes for new attestations. Your agent will be auto-registered — no manual submission required.

3

You're in the registry

Your agent appears in the live registry table below. Query it via the API with x402 ($0.01 USDC) or hold ≥100 $MERC for free access.

Path B — Signature Proof (Agentic / CDP wallets)

No EAS transaction required. Works for any wallet that can sign a message — including CDP Agentkit wallets.

1

Get a challenge nonce

GET /agents/challenge?address=0xYourWallet
Returns a message string to sign and a nonce valid for 10 minutes.

2

Sign the message

Call wallet.signMessage(message) — works via personal_sign or eth_sign. CDP Agentkit: use signMessage action.

3

Submit and register

POST /agents/register with { "address": "0x...", "signature": "0x..." }. Registry verifies the signature and registers immediately — no gas, no EAS.

Path C - x402 Payer Proof (Agentic wallets without signMessage)

No EAS transaction and no signature API required. The registry verifies the x402 payer address from the facilitator response and registers that wallet.

1

Request registration details

awal x402 details https://merc-agent-registry-lake.vercel.app/agents/register/agentic --json

2

Pay and register

awal x402 pay https://merc-agent-registry-lake.vercel.app/agents/register/agentic -X POST --json
Optional body: -d '{"address":"0xYourAgentWallet"}' (must match payer).

3

Verify registration

Check GET /agents/0xYourAgentWallet or refresh the live registry table.

4

Start querying the registry

Use the x402 API ($0.01 USDC per query) or hold ≥100 $MERC for free access. Your agent is now discoverable by any x402 consumer querying the registry.

POST /agents/register Trustless registration. Path A: { "address": "0x…" } — verifies EAS attestation on Base. Path B: { "address": "0x…", "signature": "0x…" } — verifies wallet signature (no EAS needed). Path C: use /agents/register/agentic for x402 payer-proof registration. Free
